# How to Run This Jekyll Blog Locally

## Prerequisites
- Ruby 2.6 or higher (verified: Ruby 2.6.10p210 on macOS)
- RubyGems (comes with Ruby)
- Git (for version control)

## Steps to Run Locally

### 1. Navigate to the blog directory
```bash
cd /Users/ncavaglione/IdeaProjects/fain/myblog
```

### 2. Install dependencies
```bash
bundle install
```
This installs all the required gems listed in the Gemfile, including:
- Jekyll 3.9.5
- github-pages 231
- jekyll-feed, jekyll-seo-tag, jekyll-sitemap
- And 91 total gems

### 3. Start the local server
```bash
bundle exec jekyll serve
```

### 4. Access your blog
Open your web browser and go to: http://127.0.0.1:4000/

## What We Learned

### Server Configuration
- **Source**: `/Users/ncavaglione/IdeaProjects/fain/myblog`
- **Destination**: `/Users/ncavaglione/IdeaProjects/fain/myblog/_site`
- **Auto-regeneration**: Enabled (changes to files automatically update the site)
- **Server address**: http://127.0.0.1:4000/

### Important Notes
- The server runs in the foreground - press `Ctrl+C` to stop it
- Auto-regeneration is enabled, so you can edit files and see changes immediately
- The site generates in about 0.736 seconds
- Jekyll Feed is automatically generated for posts

### Troubleshooting
- If you get "Could not locate Gemfile or .bundle/ directory", make sure you're in the correct directory (`/Users/ncavaglione/IdeaProjects/fain/myblog`)
- The Gemfile and Gemfile.lock must be present in the directory
- All dependencies are installed locally via Bundler

### File Structure
Key files for running:
- `_config.yml` - Jekyll configuration
- `Gemfile` - Ruby dependencies
- `Gemfile.lock` - Locked dependency versions
- `_posts/` - Blog posts directory
- `_layouts/` - HTML templates
- `assets/` - CSS, JS, images
